3. Theory questions
These study questions are good to get an overview of how SQL and relational databases work.

  a) What is the difference between INNER JOIN and INTERSECT?

  INTERSECT jämför rader vertikalt och skriver ut det som är exakt samma i tabellerna. INNER JOIN kopplar ihop information från två källor där värdet (ID) är samma.

  b) When are the purposes of set operations?

  Syftet är att kombinera resultaten från två eller flera SELECT-frågor till en enda lista. Man använder dem för att stapla data vertikalt (på höjden), till exempel för att slå ihop rader från olika tabeller (UNION), hitta rader som finns i båda (INTERSECT), eller filtrera bort rader som finns i den ena men inte den andra (EXCEPT) 

  c) What are the main difference between joins and set operations?

  Joins kombinerar data horisontellt genom att koppla ihop kolumner från olika tabeller baserat på en gemensam nyckel (t.ex. ID). Resultatet blir bredare rader med mer information.

  Set Operations kombinerar data vertikalt genom att stapla resultatet från två eller flera frågor på varandra. Detta kräver att frågorna har samma antal kolumner och kompatibla datatyper."

  d) When is set operators used contra logical operators?

 Logiska operatorer (AND, OR) används inuti en WHERE-sats för att filtrera rader baserat på villkor (Sant/Falskt). Man ställer krav på    datat.
 Set operatorer (UNION, INTERSECT) används mellan två hela SELECT-satser för att slå ihop eller jämföra två färdiga listor med data.

  e) How to achieve this using set operations in SQL, where A and B are result sets.

  SELECT * FROM A EXCEPT SELECT * FROM B

  f) How to achieve this using set operations in SQL, where A and B are result sets.

  (SELECT * FROM A EXCEPT SELECT * FROM B)
  UNION
  (SELECT * FROM B EXCEPT SELECT * FROM A)

  g) Does joining order matter for three or more tables?

  INNER JOIN: Nej, ordningen spelar oftast ingen roll för slutresultatet.
  * Logik: Om du kräver att data ska finnas i A, B och C, så spelar det ingen roll i vilken ordning du kollar. A+B+C är samma som C+B+A.

  LEFT JOIN / RIGHT JOIN: JA, absolut!
  * Logik: En LEFT JOIN utgår alltid från tabellen till vänster (ovanför). Om du byter ordning ändrar du vilken tabell som är "Huvudtabellen" som vi behåller alla rader ifrån.
